# Cyber Masters Course Guide

## Goal

Build a practical cyber-security skill base equivalent to a Master of Cyber Security learning path, while producing portfolio evidence Matt can use for jobs, side-income tools, or future SaaS/security products.

## Track options

### Track 1: Security Engineering
Best fit for Matt because of electronics, instrumentation, coding and practical systems background.

Focus:

- network security
- Linux and OS security
- C/assembler fundamentals
- security engineering
- cyber operations
- cloud security
- penetration testing
- advanced penetration testing
- digital forensics
- reverse engineering
- incident response
- capstone

### Track 2: Security Management & Leadership
Useful for leadership/career progression, governance, risk and compliance.

Focus:

- data security and privacy
- cybersecurity ethics
- principles of security engineering
- cyber operations
- risk and resilience
- management and governance
- threats and crime
- cyber law
- incident response
- capstone

## 24-week default plan

### Phase 1 — Foundations, weeks 1-4

- Cyber orientation
- Network security
- Linux/OS fundamentals
- Data security and privacy

Portfolio outputs:

- home network threat model
- Linux hardening checklist
- data security plan for a small business

### Phase 2 — Engineering core, weeks 5-10

- Security engineering principles
- C and assembler fundamentals
- Cloud security
- Cyber operations

Portfolio outputs:

- secure architecture review for a small SaaS
- cloud security baseline
- mini SOC playbook

### Phase 3 — Offensive/defensive practice, weeks 11-16

- Penetration testing
- Advanced penetration testing
- Digital forensics
- Reverse engineering

Portfolio outputs:

- pentest report sample
- forensics case report
- reverse engineering notes

### Phase 4 — Governance, ethics and response, weeks 17-20

- Cybersecurity ethics
- Cyber risk and resilience
- Cyber law
- Incident response

Portfolio outputs:

- responsible disclosure checklist
- risk register
- incident response plan

### Phase 5 — Capstone, weeks 21-24

Build and document a complete security project.

Recommended Matt capstone:

## Secure Small Business SaaS Stack

Secure one of Matt's own local SaaS tools:

- threat model
- authentication design
- logging plan
- backup/recovery plan
- pentest checklist
- incident response playbook
- final report

## Weekly routine

### After-work 45 minute session

1. Read one lesson topic.
2. Do one lab step.
3. Write three evidence bullets.
4. Mark module progress in the platform.

### Weekend 2-3 hour session

1. Finish one lab.
2. Write one portfolio artefact.
3. Export progress JSON.
4. Decide next weak area.

## Portfolio rule

Every module must produce evidence:

- checklist
- report
- lab notes
- diagram
- policy
- playbook
- secure design
- incident timeline
- pentest finding

No passive study only. Build proof.
